# 便利商店經營模擬（MVP）

Vite + TypeScript + Phaser 3 的網頁經營模擬遊戲。目標：在 30 個遊戲天內把資金從 $50,000 滾到 $150,000。

## 啟動

```bash
npm install
npm run dev      # 開發伺服器，瀏覽器開 http://localhost:5173
npm run build    # 打包到 dist/
npm run preview  # 預覽打包結果
```

## 玩法（核心循環）

1. **設定售價**：每項商品可調售價。漲價會降低需求、降價會提升需求（見「需求 ×係數」）。
2. **進貨**：點「🧮 建議補貨（全部）」一鍵補到預期需求量，或手動輸入數量按「進」。
3. **營業一天**：模擬整天來客消費 → 結算現金。
4. **生鮮報廢**：飯糰／便當／麵包當日賣不完會報廢（成本損失）——別囤太多。
5. **勝負**：現金達 $150,000 過關；現金 < 0 倒閉；30 天到期未達標失敗。

## 專案結構

| 路徑 | 說明 |
|------|------|
| `src/game/types.ts` | 資料模型 |
| `src/game/config.ts` | **所有可調數值與商品資料**（數值平衡改這裡） |
| `src/game/engine.ts` | 核心模擬：價格彈性、預期需求、進貨、單日結算 |
| `src/game/state.ts` | 初始狀態 |
| `src/ui/panel.ts` | DOM 管理面板（表格、按鈕、結算、勝負彈窗） |
| `src/scene/StoreScene.ts` | Phaser 像素店面場景與營業動畫 |
| `src/main.ts` | 進入點，串接邏輯／UI／場景 |

## 數值平衡（`src/game/config.ts`）

| 參數 | 預設 | 作用 |
|------|------|------|
| `START_CASH` | 45000 | 起始資金 |
| `GOAL_CASH` | 150000 | 勝利門檻 |
| `TOTAL_DAYS` | 30 | 天數上限 |
| `DAILY_FIXED_COST` | 4500 | 每日租金＋水電 |
| `BASE_CUSTOMERS` | 120 | 基準來客數 |
| `BASKET_SCALE` | 0.45 | 每位顧客平均購買件數 |
| `ELASTICITY_UP` | 1.4 | 漲價的需求衰減 |
| `ELASTICITY_DOWN` | 0.4 | 降價的需求提升 |

> 數值經模擬校準（`npm run balance`）：高手按「建議補貨」100% 過關（約第 17 天），粗心約 63%，亂囤貨必敗，且為軟性失敗（時間到未達標、零破產）。

## 數值平衡工具

```bash
npm run balance   # 用不同策略各跑 N 局，回報勝率/達標日/破產率
```
調整 `src/game/config.ts` 後重跑即可看難度變化。
